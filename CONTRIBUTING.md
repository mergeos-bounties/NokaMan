# Contributing to NokaMan

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/NokaMan.git`
3. Create a feature branch: `git checkout -b feat/your-feature`

## Development Setup

```bash
# Install dependencies (adjust based on your package manager)
npm install  # or poetry install, cargo build, etc.
```

## Code Style

- Run `ruff format` before committing
- Run `ruff check` to lint
- Ensure all tests pass: `pytest` or equivalent

## Submitting Changes

1. Push your branch: `git push origin feat/your-feature`
2. Open a Pull Request against the `main` branch
3. In the PR description, explain what the change does and why

## Reporting Issues

- Check existing issues before filing a duplicate
- Use the issue templates if available
- Include steps to reproduce for bugs

## First Time Contributors

Look for issues labeled `good first issue` — they are specifically chosen to be accessible for newcomers.
